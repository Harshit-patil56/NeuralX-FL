SECTION 4: ASYNCHRONOUS FL - FULL IMPLEMENTATION SPECIFICATION
================================================================================

PROBLEM WITH SYNCHRONOUS FL:
- Server waits for ALL selected clients before aggregating
- If 1 client is slow (straggler) the entire round is blocked
- If 1 client drops out the round fails or must restart
- In real systems (mobile, hospital networks), this is impractical

WHAT ASYNC FL DOES:
- Server maintains a global version counter (global_round)
- Server maintains an update buffer (queue of received updates)
- Clients train independently and submit whenever they finish
- Server aggregates as updates arrive -- does not wait for all clients
- Two async triggering strategies:
  a) Threshold-based: aggregate when buffer has K updates (e.g., K=4 out of 10)
  b) Time-based: aggregate after T seconds regardless of how many arrived

ASYNC FL ALGORITHM (implement this exactly):

  Initialize:
    global_model = initial_parameters
    global_round = 0
    update_buffer = []
    staleness_map = {}  # tracks which global_round each client trained on

  For each client submission (client_id, parameters, num_examples, client_round):
    staleness = global_round - client_round  # how stale is this update
    staleness_weight = 1.0 / (1.0 + staleness)  # downweight stale updates
    update_buffer.append({
        'params': parameters,
        'num_examples': num_examples,
        'staleness': staleness,
        'weight': staleness_weight,
        'client_id': client_id
    })

    if len(update_buffer) >= ASYNC_BUFFER_SIZE:
        filtered_updates = malicious_filter(update_buffer)
        new_params = robust_aggregate(filtered_updates)
        global_model = new_params
        global_round += 1
        update_buffer = []

STALENESS WEIGHTING:
  - Fresh update (staleness=0): weight = 1.0
  - 1 round stale: weight = 0.5
  - 2 rounds stale: weight = 0.33
  - Formula: weight = 1 / (1 + staleness)
  - Multiply staleness_weight by num_examples for final weighted average

SIMULATING ASYNC IN FLOWER SIMULATION:
  Flower's run_simulation is synchronous by design.
  To simulate async behavior within Flower's framework:
  - Use fit_config to assign each client a random "delay" value
  - In the custom Strategy.aggregate_fit(), sort results by simulated arrival time
  - Process results in arrival order, not submission order
  - CRITICAL ORDER: filter ALL results for malicious/noisy content FIRST,
    THEN select the first ASYNC_BUFFER_SIZE clean updates that "arrive"
  - Filtering after selection causes attacker-heavy buffers when attackers respond
    fast. Filtering before selection ensures the async buffer is filled only by
    clean or lightly-noisy updates.
  This correctly simulates async behavior within a synchronous simulation engine.

VERSIONED GLOBAL MODEL TRACKING (fixes Issues 1 and 3):
  Because Flower simulation is synchronous, true per-client model versioning
  is simulated as follows:
  - The strategy stores self.global_version (incremented after each aggregation).
  - configure_fit() sends 'model_version': self.global_version in each client config.
  - client fit() returns 'client_model_version' in metadata.
  - staleness = self.global_version - client_model_version
  - This makes staleness reflect the model version gap, not just the round number.
  - For the simulation's purposes this is equivalent: each Flower round IS one
    global model version. The version counter makes the semantic explicit and
    allows future extension where multiple aggregations happen per Flower round.

CONCRETE ASYNC SIMULATION IN STRATEGY (CORRECTED ORDER):

  def aggregate_fit(self, server_round, results, failures):
      if not results:
          return self.current_parameters, {'error': 'no_results'}

      # STEP 1: Build ALL update dicts from ALL results (before any selection)
      all_updates = []
      for proxy, res in results:
          client_ver = res.metrics.get('client_model_version', self.global_version)
          staleness = self.global_version - client_ver
          staleness_weight = 1.0 / (1 + staleness)
          all_updates.append({
              'params': parameters_to_ndarrays(res.parameters),
              'num_examples': res.num_examples,
              'staleness_weight': staleness_weight,
              'client_id': int(proxy.cid),
              'is_malicious': res.metrics.get('is_malicious', False),
              'simulated_delay': res.metrics.get('simulated_delay', 0.0),
          })

      # STEP 2: Apply detection filters on ALL results (not just the fast ones)
      all_updates, norm_flagged  = filter_by_norm(all_updates, self.norm_threshold)
      all_updates, cosine_flagged = filter_by_cosine(all_updates, self.cosine_threshold)
      all_flagged = norm_flagged + cosine_flagged

      # STEP 3: THEN sort survivors by simulated arrival time and take buffer_size
      all_updates_sorted = sorted(all_updates, key=lambda u: u['simulated_delay'])
      async_updates = all_updates_sorted[:self.async_buffer_size]

      if not async_updates:
          return self.current_parameters, {'flagged': len(all_flagged)}

      # STEP 4: Robust aggregation on clean, async-selected updates
      aggregated = aggregate_robust(
          [u['params'] for u in async_updates],
          method=self.aggregation_method,
          weights=[u['staleness_weight'] * u['num_examples'] for u in async_updates],
          trim_fraction=self.trim_fraction,
      )

      self.current_parameters = ndarrays_to_parameters(aggregated)
      self.global_version += 1   # increment versioned model counter
      return self.current_parameters, {
          'async_clients_used': len(async_updates),
          'total_submitted': len(results),
          'clients_filtered': len(all_flagged),
          'flagged_ids': str(all_flagged),
          'global_version': self.global_version,
      }

SECTION 5: ROBUST AGGREGATION ALGORITHMS - FULL IMPLEMENTATION
================================================================================

WHY FEDAVG FAILS UNDER ATTACK:
- FedAvg takes a weighted mean of all parameters
- One malicious client sending large scaled gradients can dominate the mean
- Result: global model corrupted, accuracy drops to near 0%

ALGORITHM 1: TRIMMED MEAN (implement this as primary)
  - For each parameter dimension independently:
  - Sort the values from all clients
  - Remove top beta% and bottom beta% (beta=0.1 means remove 10% each side)
  - Average the remaining values
  - Robust against up to beta fraction of Byzantine clients

  def trimmed_mean(layer_updates, trim_fraction=0.1):
      # layer_updates: list of 1D numpy arrays, one per client
      stacked = np.stack(layer_updates, axis=0)  # shape: (num_clients, num_params)
      n = len(layer_updates)
      k = int(np.floor(trim_fraction * n))

      if k == 0:
          return np.mean(stacked, axis=0)

      # Guard: if trim removes all elements (trim_fraction >= 0.5), fall back to median
      if n - 2 * k <= 0:
          return np.median(stacked, axis=0)

      sorted_stacked = np.sort(stacked, axis=0)
      trimmed = sorted_stacked[k:n-k, :]
      return np.mean(trimmed, axis=0)

ALGORITHM 2: COORDINATE-WISE MEDIAN (implement as secondary)
  - For each parameter dimension independently, take the median across clients
  - More robust than mean against outliers
  - No trim_fraction parameter needed

  def coordinate_median(layer_updates):
      stacked = np.stack(layer_updates, axis=0)
      return np.median(stacked, axis=0)

ALGORITHM 3: KRUM (optional, implement for bonus points)
  - Selects the single update closest to its n-f-2 nearest neighbors
  - f = number of assumed Byzantine clients
  - Multi-Krum: select m updates with lowest Krum scores, then average

  def krum(layer_updates, num_byzantine, multi=False, m=None):
      n = len(layer_updates)
      f = num_byzantine
      scores = []
      for i in range(n):
          distances = sorted([
              np.sum((layer_updates[i] - layer_updates[j]) ** 2)
              for j in range(n) if i != j
          ])
          score = sum(distances[:n - f - 2])
          scores.append(score)

      if multi and m is not None:
          selected = [layer_updates[i] for i in np.argsort(scores)[:m]]
          return np.mean(np.stack(selected, axis=0), axis=0)
      else:
          return layer_updates[np.argmin(scores)]

APPLY PER LAYER (wrapper function):

  def aggregate_robust(all_client_params, method='trimmed_mean', weights=None, **kwargs):
      # all_client_params: List[List[np.ndarray]] -- outer=clients, inner=layers
      num_layers = len(all_client_params[0])
      aggregated = []

      for layer_idx in range(num_layers):
          layer_updates = [
              client_params[layer_idx].flatten()
              for client_params in all_client_params
          ]

          if method == 'trimmed_mean':
              agg_layer = trimmed_mean(layer_updates, **kwargs)
          elif method == 'median':
              agg_layer = coordinate_median(layer_updates)
          elif method == 'krum':
              agg_layer = krum(layer_updates, **kwargs)
          elif method == 'fedavg':
              # Weighted mean (standard FedAvg baseline)
              if weights is not None:
                  w = np.array(weights, dtype=np.float32)
                  w = w / w.sum()
                  stacked = np.stack(layer_updates, axis=0)
                  agg_layer = np.sum(stacked * w[:, np.newaxis], axis=0)
              else:
                  agg_layer = np.mean(np.stack(layer_updates, axis=0), axis=0)
          else:
              raise ValueError(f"Unknown aggregation method: {method}")

          original_shape = all_client_params[0][layer_idx].shape
          aggregated.append(agg_layer.reshape(original_shape).astype(np.float32))

      return aggregated

PART A: SIMULATING MALICIOUS CLIENTS

  Types of attacks to simulate:

  1. SCALING ATTACK: multiply gradients by large factor (10x or 100x)
     Effect on FedAvg: catastrophic -- pulls global model in malicious direction

  2. RANDOM NOISE ATTACK: send completely random parameters N(0,1)
     Effect: degrades model proportional to fraction of attackers

  3. SIGN FLIP ATTACK: negate all parameters (multiply by -1)
     Designed to directly oppose gradient descent

  4. NOISY CLIENT (DATA QUALITY ISSUE): trains on data with 30% label noise.
     Labels randomly flipped to a wrong class during data loading.
     NOT adversarial intent -- simulates real-world mislabeled data:
     hospital records with annotation errors, mobile auto-labeling mistakes,
     or crowdsourced datasets with noisy workers.
     Effect: update points in a subtly wrong direction, degrades convergence
     gradually and cumulatively -- much milder than adversarial attacks.
     Detection: cosine similarity filter catches updates that diverge from
     the consensus direction across rounds.
     NOISY_CLIENT_IDS = {4, 5}  # 20% noisy (distinct from malicious)
     In configure_fit(), add to config: 'noise_rate': 0.3 if noisy else 0.0
     In load_data(), apply label flipping when noise_rate > 0 (see Section 10).

  5. UNRELIABLE CLIENT (DROPOUT / TIMEOUT): randomly fails to respond per round.
     Simulates mobile devices going offline, hospital network timeouts,
     or flaky edge nodes that disconnect mid-training.
     Effect: reduces the number of updates in the async buffer; the async
     aggregation design naturally tolerates this (buffer threshold, not quorum).
     Implementation: in client fit(), if random.random() < dropout_prob,
     raise RuntimeError. Flower catches it and adds the client to the
     failures list in aggregate_fit(). No special handling needed.
     UNRELIABLE_CLIENT_IDS = {6, 7}  # 20% unreliable (distinct from malicious/noisy)
     In configure_fit(), add: 'dropout_prob': 0.4 if unreliable else 0.0

  Inject attack in client fit() method (CANONICAL version -- matches Section 11):

  def fit(self, parameters, config):

          raise RuntimeError(f"Client {self.partition_id}: simulated dropout/timeout")

      set_weights(self.net, parameters)
      train(self.net, self.trainloader, epochs=int(config.get('local_epochs', 2)))
      updated_params = get_weights(self.net)

      is_malicious = bool(config.get('is_malicious', False))
      attack_type = str(config.get('attack_type', 'none'))

      if is_malicious:
          if attack_type == 'scaling':
              scale = float(config.get('attack_scale', 10.0))
              updated_params = [p * scale for p in updated_params]
          elif attack_type == 'random':
              updated_params = [
                  np.random.randn(*p.shape).astype(np.float32)
                  for p in updated_params
              ]
          elif attack_type == 'sign_flip':
              updated_params = [-p for p in updated_params]

      simulated_delay = float(np.random.exponential(
          scale=float(config.get('delay_scale', 1.0))

      return updated_params, len(self.trainloader.dataset), {
          'client_id': self.partition_id,
          'is_malicious': is_malicious,
          'client_round': int(config.get('current_round', 0)),
          # Echo back the model version this client trained on (used for staleness)
          'client_model_version': int(config.get('model_version', 0)),
          'simulated_delay': simulated_delay,

PART B: MALICIOUS CLIENT DETECTION METHODS

  METHOD 1: L2 NORM FILTERING (catches scaling attacks)
  - Compute L2 norm of each client's flattened parameter update
  - Compute median norm across all clients
  - Reject clients whose norm exceeds threshold_multiplier * median_norm

  def filter_by_norm(updates, threshold_multiplier=3.0):
      for update in updates:
          flat = np.concatenate([p.flatten() for p in update['params']])
          update['norm'] = float(np.linalg.norm(flat))

      norms = [u['norm'] for u in updates]
      median_norm = float(np.median(norms))

      clean_updates = []
      flagged = []
      for update in updates:
          if update['norm'] <= threshold_multiplier * median_norm:
              clean_updates.append(update)
          else:
              flagged.append(update['client_id'])

      return clean_updates, flagged

  METHOD 2: COSINE SIMILARITY FILTERING (catches sign-flip and directional attacks)
  - Use coordinate-wise MEDIAN of all updates as the reference direction,
    NOT the mean. Mean is attacker-sensitive; if attackers dominate the buffer
    the mean direction shifts toward the attack. Median is robust.
  - Compute cosine similarity of each update with the median direction.
  - Reject clients with cosine similarity below threshold (pointing wrong direction).
  - This also reliably catches noisy clients whose updates consistently diverge
    from the consensus direction of honest clients.

  def filter_by_cosine(updates, similarity_threshold=0.0):
      if len(updates) < 2:
          return updates, []

      flat_updates = [
          np.concatenate([p.flatten() for p in u['params']])
          for u in updates
      ]
      # Use coordinate-wise MEDIAN as robust reference direction (not mean)
      stacked = np.stack(flat_updates, axis=0)    # shape: (n_clients, n_params)
      median_update = np.median(stacked, axis=0)  # robust to outlier directions
      median_norm = float(np.linalg.norm(median_update))

      if median_norm < 1e-10:
          return updates, []  # degenerate case: all updates near zero

      clean_updates = []
      flagged = []
      for i, update in enumerate(updates):
          flat = flat_updates[i]
          cosine_sim = float(
              np.dot(flat, median_update) /
              (np.linalg.norm(flat) * median_norm + 1e-10)
          )
          update['cosine_sim'] = cosine_sim

          if cosine_sim >= similarity_threshold:
              clean_updates.append(update)
          else:
              flagged.append(update['client_id'])

      return clean_updates, flagged

  PIPELINE: Apply norm filter FIRST (removes scaling attacks), then cosine filter
  on survivors using median reference (removes sign-flip, directional, noisy clients).
  Log all flagged client IDs per round for evaluation metrics.
  Track true positive rate: were flagged clients actually in MALICIOUS_CLIENT_IDS
  or NOISY_CLIENT_IDS?

          self.global_version = 0
          # Dropout tracking: logs how many clients failed per round (Issue 5)
          self.dropout_history = []  # list of {'round': r, 'dropped': n, 'submitted': n}

      def initialize_parameters(self, client_manager):
          return self.initial_parameters

      def configure_fit(self, server_round, parameters, client_manager):
          available = client_manager.num_available()
          sample_size = min(self.num_clients_per_round, available)
          clients = client_manager.sample(num_clients=sample_size)

          fit_configurations = []
          for client in clients:
              cid = int(client.cid)
              is_malicious  = cid in self.malicious_client_ids
              is_noisy      = cid in self.noisy_client_ids
              is_unreliable = cid in self.unreliable_client_ids
              config = {
                  'local_epochs': 2 if server_round <= 3 else 5,
                  'current_round': server_round,
                  'model_version': self.global_version,  # client returns this for staleness
                  'delay_scale': 2.0,
                  # Adversarial client
                  'is_malicious': is_malicious,
                  'attack_type': 'scaling' if is_malicious else 'none',
                  'attack_scale': 10.0,
                  # Noisy client (data quality)
                  'is_noisy': is_noisy,
                  'noise_rate': 0.3 if is_noisy else 0.0,
                  # Unreliable client (dropout)
                  'dropout_prob': 0.4 if is_unreliable else 0.0,
              }
              fit_configurations.append((client, FitIns(parameters, config)))
          return fit_configurations

      def aggregate_fit(self, server_round, results, failures):
          # Log unreliable client dropout (Issue 5: must be demonstrated, not assumed)
          self.dropout_history.append({
              'round': server_round,
              'dropped': len(failures),
              'submitted': len(results),
              'total_selected': len(results) + len(failures),
          })

          if not results:
              return self.current_parameters, {'error': 'no_results', 'dropped': len(failures)}

          # STEP 1: Build update dicts from ALL submitted results
          all_updates = []
          for proxy, res in results:
              client_ver = res.metrics.get('client_model_version', self.global_version)
              staleness = self.global_version - client_ver
              staleness_weight = 1.0 / (1 + staleness)
              all_updates.append({
                  'params': parameters_to_ndarrays(res.parameters),
                  'num_examples': res.num_examples,
                  'staleness_weight': staleness_weight,
                  'client_id': int(proxy.cid),
                  'is_malicious': res.metrics.get('is_malicious', False),
                  'simulated_delay': res.metrics.get('simulated_delay', 0.0),
              })

          # STEP 2: Filter ALL results BEFORE async buffer selection (Issue 2)
          all_updates, norm_flagged   = filter_by_norm(all_updates, self.norm_threshold)
          all_updates, cosine_flagged = filter_by_cosine(all_updates, self.cosine_threshold)
          all_flagged = norm_flagged + cosine_flagged
          self.flagged_history.append({'round': server_round, 'flagged': all_flagged})

          # STEP 3: THEN sort clean survivors by simulated delay, take buffer_size
          all_updates_sorted = sorted(all_updates, key=lambda u: u['simulated_delay'])
          async_updates = all_updates_sorted[:self.async_buffer_size]

          if not async_updates:
              return self.current_parameters, {
                  'flagged': len(all_flagged),
                  'dropped': len(failures),
              }

          # STEP 4: Robust aggregation on clean, async-selected updates
          aggregated = aggregate_robust(
              [u['params'] for u in async_updates],
              method=self.aggregation_method,
              weights=[u['staleness_weight'] * u['num_examples'] for u in async_updates],
              trim_fraction=self.trim_fraction,
          )

          self.current_parameters = ndarrays_to_parameters(aggregated)
          self.global_version += 1  # advance versioned model counter

          metrics = {
              'async_clients_used': len(async_updates),
              'total_submitted': len(results),
              'clients_filtered': len(all_flagged),
              'clients_dropped': len(failures),
              'flagged_ids': str(all_flagged),
              'global_version': self.global_version,
          }
          self.round_metrics.append(metrics)
          return self.current_parameters, metrics

      def configure_evaluate(self, server_round, parameters, client_manager):
          return []  # Server-side evaluation only

      def aggregate_evaluate(self, server_round, results, failures):
          return None, {}

      def evaluate(self, server_round, parameters):
          if self.evaluate_fn is None:
              return None
          return self.evaluate_fn(
              server_round,
              parameters_to_ndarrays(parameters),
              {}
          )

  # Model: PathologyNet
  # Input : 3-channel RGB 28x28 pathology slide patches (PathMNIST format)
  # Output: 9-class tissue classification
  #   Class 0: adipose tissue        Class 5: normal colon mucosa
  #   Class 1: background            Class 6: colorectal adenocarcinoma
  #   Class 2: debris                Class 7: smooth muscle
  #   Class 3: lymphocytes           Class 8: stroma
  #   Class 4: mucus

  import torch
  import torch.nn as nn
  import torch.nn.functional as F

  class PathologyNet(nn.Module):
      def __init__(self):
          super().__init__()
          self.conv1 = nn.Conv2d(3, 32, 3, padding=1)  # 3-channel RGB input
          self.conv2 = nn.Conv2d(32, 64, 3, padding=1)
          self.pool = nn.MaxPool2d(2, 2)
          self.fc1 = nn.Linear(64 * 7 * 7, 128)
          self.fc2 = nn.Linear(128, 9)               # 9 tissue classes
          self.dropout = nn.Dropout(0.25)

      def forward(self, x):
          x = self.pool(F.relu(self.conv1(x)))  # 28x28 -> 14x14
          x = self.pool(F.relu(self.conv2(x)))  # 14x14 -> 7x7
          x = x.view(-1, 64 * 7 * 7)
          x = F.relu(self.fc1(x))
          x = self.dropout(x)
          return self.fc2(x)

  def get_weights(net):
      return [val.cpu().numpy() for _, val in net.state_dict().items()]

  def set_weights(net, parameters):
      params_dict = zip(net.state_dict().keys(), parameters)
      state_dict = {k: torch.tensor(v) for k, v in params_dict}
      net.load_state_dict(state_dict, strict=True)

  DEVICE = torch.device('cuda' if torch.cuda.is_available() else 'cpu')

  def train(net, trainloader, epochs, device=None):
      device = device or DEVICE
      net.to(device)
      net.train()
      optimizer = torch.optim.Adam(net.parameters(), lr=0.001)
      criterion = nn.CrossEntropyLoss()
      for _ in range(epochs):
          for batch in trainloader:
              images = batch['image'] if isinstance(batch, dict) else batch[0]
              labels = batch['label'] if isinstance(batch, dict) else batch[1]
              images, labels = images.to(device), labels.to(device)
              optimizer.zero_grad()
              criterion(net(images), labels).backward()
              optimizer.step()

  def evaluate_model(net, testloader, device=None):
      device = device or DEVICE
      net.to(device)
      net.eval()
      correct, total, loss_sum = 0, 0, 0.0
      criterion = nn.CrossEntropyLoss()
      with torch.no_grad():
          for batch in testloader:
              images = batch['image'] if isinstance(batch, dict) else batch[0]
              labels = batch['label'] if isinstance(batch, dict) else batch[1]
              images, labels = images.to(device), labels.to(device)
              outputs = net(images)
              loss_sum += criterion(outputs, labels).item()
              _, predicted = torch.max(outputs, 1)
              total += labels.size(0)
              correct += (predicted == labels).sum().item()
      return correct / total, loss_sum / len(testloader)

  # PathMNIST: real colon tissue pathology images, 28x28 RGB, 9 classes.
  # Source paper: "MedMNIST v2: A Large-Scale Lightweight Benchmark for
  #   2D and 3D Biomedical Image Classification" (Yang et al., 2023)
  # Install: pip install medmnist
  # Data: ~100k slides. Downloads once to ./data on first call.

  import medmnist
  from medmnist import PathMNIST
  import numpy as np
  import torch
  from torch.utils.data import DataLoader, TensorDataset
  from torchvision.transforms import Compose, ToTensor, Normalize

  NUM_CLIENTS = 10
  NUM_CLASSES = 9
  DIRICHLET_ALPHA = 0.5  # Lower = more heterogeneous. 0.5 is standard in FL literature.

  # PathMNIST channel statistics (pre-computed, matches medmnist official values)
  PATHMNIST_MEAN = (0.7405, 0.5330, 0.7058)
  PATHMNIST_STD  = (0.1270, 0.1770, 0.1256)
  PATHMNIST_TRANSFORM = Compose([ToTensor(), Normalize(PATHMNIST_MEAN, PATHMNIST_STD)])

  # ---- In-memory cache: download once, reuse across all clients ----
  _PATHMNIST_CACHE: dict = {}

  def _get_pathmnist_tensors(split: str):
      """Load and cache PathMNIST for a given split ('train' or 'test').
      Returns (images: Tensor[N,3,28,28], labels: Tensor[N])."""
      if split in _PATHMNIST_CACHE:
          return _PATHMNIST_CACHE[split]
      dataset = PathMNIST(split=split, transform=PATHMNIST_TRANSFORM,
                          download=True, root='./data')
      images, labels = [], []
      for img, lbl in dataset:
          images.append(img)
          labels.append(int(lbl.item() if hasattr(lbl, 'item') else lbl[0]))
      result = (torch.stack(images), torch.tensor(labels, dtype=torch.long))
      _PATHMNIST_CACHE[split] = result
      return result

  def _dirichlet_split(labels: torch.Tensor, num_clients: int,
                        alpha: float, seed: int = 42):
      """Partition indices into num_clients subsets via Dirichlet distribution.
      Returns List[List[int]] where list[i] = sample indices for hospital i."""
      rng = np.random.default_rng(seed)
      n_classes = int(labels.max().item()) + 1
      class_idx = [np.where(labels.numpy() == c)[0] for c in range(n_classes)]
      client_idx = [[] for _ in range(num_clients)]
      for c_idx in class_idx:
          rng.shuffle(c_idx)
          props = rng.dirichlet(np.repeat(alpha, num_clients))
          props = (props * len(c_idx)).astype(int)
          props[-1] = len(c_idx) - props[:-1].sum()   # ensure no rounding loss
          for cid, sl in enumerate(np.split(c_idx, np.cumsum(props[:-1]))):
              client_idx[cid].extend(sl.tolist())
      return client_idx

  def add_label_noise(labels: torch.Tensor, noise_rate: float,
                       num_classes: int = NUM_CLASSES) -> torch.Tensor:
      """Randomly flip noise_rate fraction of labels to a different class.
      Simulates labelling errors in a hospital (inter-annotator disagreement)."""
      if noise_rate <= 0.0:
          return labels
      noisy = labels.clone()
      n_noisy = int(len(labels) * noise_rate)
      noisy_idx = torch.randperm(len(labels))[:n_noisy]
      for i in noisy_idx:
          original = noisy[i].item()
          wrong = (original + torch.randint(1, num_classes, (1,)).item()) % num_classes
          noisy[i] = wrong
      return noisy

  def load_data(partition_id: int, num_clients: int = NUM_CLIENTS,
                noise_rate: float = 0.0, dirichlet_alpha: float = DIRICHLET_ALPHA):
      """Return (trainloader, testloader) for hospital `partition_id`.
      dirichlet_alpha=0.5  -> non-IID (default)
      dirichlet_alpha=1000 -> near-IID (for Exp E IID variant)"""
      train_imgs, train_labels = _get_pathmnist_tensors('train')
      test_imgs,  test_labels  = _get_pathmnist_tensors('test')

      train_idx = _dirichlet_split(train_labels, num_clients, dirichlet_alpha, seed=42)
      test_idx  = _dirichlet_split(test_labels,  num_clients, dirichlet_alpha, seed=43)

      t_imgs   = train_imgs[train_idx[partition_id]]
      t_labels = train_labels[train_idx[partition_id]]
      v_imgs   = test_imgs[test_idx[partition_id]]
      v_labels = test_labels[test_idx[partition_id]]

      if noise_rate > 0.0:
          t_labels = add_label_noise(t_labels, noise_rate)

      trainloader = DataLoader(TensorDataset(t_imgs, t_labels),
                               batch_size=32, shuffle=True)
      testloader  = DataLoader(TensorDataset(v_imgs, v_labels), batch_size=32)
      return trainloader, testloader

  def load_global_test():
      """Full PathMNIST test set for server-side global evaluation."""
      test_imgs, test_labels = _get_pathmnist_tensors('test')
      return DataLoader(TensorDataset(test_imgs, test_labels),
                        batch_size=64, shuffle=False)

  import flwr as fl
  import numpy as np
  from flwr.common import NDArrays, Scalar
  from typing import Dict, Tuple

  class AsyncFLClient(fl.client.NumPyClient):

      def __init__(self, partition_id: int, net, trainloader, testloader):
          self.partition_id = partition_id
          self.net = net
          self.trainloader = trainloader
          self.testloader = testloader

      def fit(self, parameters: NDArrays, config: Dict[str, Scalar]):
          # Simulate client dropout / network timeout BEFORE training
          dropout_prob = float(config.get('dropout_prob', 0.0))
          if dropout_prob > 0.0 and np.random.random() < dropout_prob:
              raise RuntimeError(f"Client {self.partition_id}: simulated dropout/timeout")

          set_weights(self.net, parameters)
          train(self.net, self.trainloader, epochs=int(config.get('local_epochs', 2)))
          updated_params = get_weights(self.net)

          is_malicious = bool(config.get('is_malicious', False))
          attack_type = str(config.get('attack_type', 'none'))

          if is_malicious:
              if attack_type == 'scaling':
                  scale = float(config.get('attack_scale', 10.0))
                  updated_params = [p * scale for p in updated_params]
              elif attack_type == 'random':
                  updated_params = [
                      np.random.randn(*p.shape).astype(np.float32)
                      for p in updated_params
                  ]
              elif attack_type == 'sign_flip':
                  updated_params = [-p for p in updated_params]

          simulated_delay = float(
              np.random.exponential(scale=float(config.get('delay_scale', 1.0)))
          )

          return updated_params, len(self.trainloader.dataset), {
              'client_id': self.partition_id,
              'is_malicious': is_malicious,
              'client_round': int(config.get('current_round', 0)),
              # Echo back the model version this client trained on.
              # Strategy uses this to compute staleness = global_version - client_model_version
              'client_model_version': int(config.get('model_version', 0)),
              'simulated_delay': simulated_delay,
          }

      def evaluate(self, parameters: NDArrays, config: Dict[str, Scalar]):
          set_weights(self.net, parameters)
          accuracy, loss = evaluate_model(self.net, self.testloader)
          return float(loss), len(self.testloader.dataset), {
              'accuracy': float(accuracy)
          }


  def client_fn(context):
      partition_id = int(context.node_config['partition-id'])
      # Hospital identity: Hospital-0 through Hospital-9
      # Read dirichlet_alpha from run_config (allows IID experiment override)
      alpha = float(context.run_config.get('dirichlet_alpha', DIRICHLET_ALPHA))
      # Noisy hospitals: 4 and 5 have 30% label noise (labelling disagreement)
      noise_rate = 0.3 if partition_id in {4, 5} else 0.0
      net = PathologyNet()
      trainloader, testloader = load_data(
          partition_id, noise_rate=noise_rate, dirichlet_alpha=alpha
      )
      return AsyncFLClient(partition_id, net, trainloader, testloader).to_client()

  def compute_attack_metrics(acc_clean, acc_attacked, acc_defended):
      """Quantify attack success and defense recovery (Issue 8)."""
      attack_damage    = [c - a for c, a in zip(acc_clean, acc_attacked)]
      defense_recovery = [d - a for d, a in zip(acc_defended, acc_attacked)]
      return {
          'avg_attack_damage':    float(np.mean(attack_damage)),
          'max_attack_damage':    float(np.max(attack_damage)),
          'avg_defense_recovery': float(np.mean(defense_recovery)),
          'final_clean_acc':      float(acc_clean[-1]),
          'final_attacked_acc':   float(acc_attacked[-1]),
          'final_defended_acc':   float(acc_defended[-1]),
      }

  def print_summary(experiment_name, accuracy_history, flagged_history,
                    dropout_history, malicious_ids, noisy_ids):
      all_bad_ids = malicious_ids | noisy_ids
      final_acc = accuracy_history[-1]
      convergence_round = next(
          (i+1 for i, a in enumerate(accuracy_history) if a >= 0.90),
          None
      )
      total_flagged = sum(len(e['flagged']) for e in flagged_history)
      true_positives = sum(
          len(set(e['flagged']) & all_bad_ids) for e in flagged_history
      )
      detection_rate = true_positives / (len(all_bad_ids) * len(flagged_history) + 1e-10)
      total_dropped  = sum(e['dropped'] for e in dropout_history)
      total_selected = sum(e['total_selected'] for e in dropout_history)
      dropout_rate   = total_dropped / (total_selected + 1e-10)

      print(f"\n{'='*50}")
      print(f"Experiment: {experiment_name}")
      print(f"  Final Accuracy:      {final_acc:.4f}")
      print(f"  Convergence Round:   {convergence_round or 'Did not reach 90%'}")
      print(f"  Total Flagged:       {total_flagged}")
      print(f"  Avg Detection Rate:  {detection_rate:.2%}")
      print(f"  Client Dropout Rate: {dropout_rate:.2%}")
      print(f"  Total Dropped:       {total_dropped} / {total_selected} selections")
      print(f"{'='*50}\n")

SECTION 18: TECHNICAL CONSTRAINTS AND WARNINGS (DO NOT SKIP)
================================================================================

1. FLOWER SIMULATION IS SYNCHRONOUS INTERNALLY
   run_simulation does not run clients in true parallel threads.
   Async behavior must be SIMULATED via metadata (simulated_delay in metrics).
   The Strategy.aggregate_fit() is where all async logic lives.
   Do not try to spawn threads inside client fit(). Flower manages execution.

2. NUMPY DTYPE CONSISTENCY
   Always cast aggregated arrays back to float32: agg.astype(np.float32)
   Mixed dtypes (float32 vs float64) cause set_weights() to silently corrupt weights.

3. PARAMETER SERIALIZATION
   parameters_to_ndarrays(parameters) -> List[np.ndarray]
   ndarrays_to_parameters(ndarrays) -> Parameters
   Always use these conversion functions. Never manipulate Parameters objects directly.

4. EMPTY RESULTS AFTER FILTERING
   If all clients are filtered as malicious, aggregate_fit must still return something.
   Store self.current_parameters in the strategy and return it unchanged if all filtered.

5. CLIENT METADATA TYPES
   metrics dict values must be Scalar types: int, float, str, bool.
   Do not put lists or numpy arrays directly in metrics dict.
   Convert lists to str if needed: str([1, 2, 3])

6. REPRODUCIBILITY
   Set seeds: np.random.seed(SEED), torch.manual_seed(SEED)
   Pass seed=42 to DirichletPartitioner.
   Flower simulation scheduling is non-deterministic -- note this in README.
   For more deterministic results: set num_cpus=1 and num_gpus=0.0

7. DP WRAPPER INTERACTION
   DifferentialPrivacyClientSideAdaptiveClipping wraps your strategy object.
   It intercepts aggregate_fit and adds noise AFTER your aggregation runs.
   Your strategy's aggregate_fit must return clean ndarrays_to_parameters output.
   The adaptiveclipping_mod on the client side clips BEFORE returning parameters.

8. BATCH FORMAT FROM FLOWER DATASETS
   Flower Datasets (HuggingFace backend) returns batches as dicts: {'image': ..., 'label': ...}
   Torchvision MNIST returns tuples: (image, label)
   Handle both in train() and evaluate_model() functions.

9. CONFIGURE_FIT RETURN FORMAT
   Must return List[Tuple[ClientProxy, FitIns]] -- not a plain list of configs.
   FitIns takes (parameters, config_dict) as arguments.

10. STRATEGY MUST TRACK CURRENT PARAMETERS ITSELF
    Flower does not give you the current global parameters in aggregate_fit.
    Store self.current_parameters = ndarrays_to_parameters(aggregated) after each round.
    Return self.current_parameters as fallback when all updates are rejected.

================================================================================
SECTION 19: WHAT MAKES THIS INDUSTRY-STANDARD (DESIGN RATIONALE)
================================================================================

1. MODULAR AGGREGATION ENGINE
   Aggregation algorithms are standalone pure functions (no class state).
   Strategy takes aggregation_method as a string parameter.
   Easy to swap algorithms without touching the strategy or client code.

2. STALENESS-AWARE WEIGHTING
   Updates from stale clients contribute less: weight = 1 / (1 + staleness).
   Prevents outdated gradients from corrupting the global model.
   Standard approach from FedAsync paper (Xie et al., 2019).

3. LAYERED BYZANTINE DEFENSE
   Norm filter catches scaling attacks.
   Cosine filter catches sign-flip and directional poisoning.
   Robust aggregation (trimmed mean) handles residual Byzantine influence.
   Defense-in-depth -- no single point of failure.

4. DIFFERENTIAL PRIVACY WITH ADAPTIVE CLIPPING
   Adaptive clipping adjusts the clipping norm each round dynamically.
   Better than fixed clipping -- avoids over-penalizing benign updates.
   Formally trackable privacy budget (epsilon, delta).

5. BUFFER-BASED ASYNC
   Aggregation triggers when buffer hits threshold, not when all clients respond.
   System never blocks waiting for stragglers.
   Any responding clients contribute to the next model update.

6. SEPARATION OF CONCERNS
   data.py, model.py, client.py, strategy.py, aggregation.py, detection.py are independent.
   Each file can be understood, tested, and replaced independently.
